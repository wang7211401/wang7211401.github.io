promise 是什么

Promise 对象用于一个异步操作的最终完成（或失败）及其结果值的表示。（简单说就是处理异步请求。一个诺言，一个成功，一个失败。）

promise 状态

一个Promise必须处在其中之一的状态：pending, fulfilled 或 rejected.

pending 状态：可以转换到 fulfilled 或 rejected 状态。
fulfilled 状态：不能转换成任何其它状态；必须有一个值，且这个值不能被改变。
rejected 状态：不能转换成任何其它状态；必须有一个值，且这个值不能被改变。

promise 对象

创建 promise 对象

1.new Promise(fn) 返回一个 promise 对象。

2.fn是个函数，fn有两个参数

  • resolve：处理结果成功的时候调用resolve(处理结果值)。
  • reject：处理结果错误的时候调用reject(Error对象)。

then 方法

一个 Promise 必须提供一个 then 方法来获取其值。

Promise 的 then 方法接受两个参数：

promise.then(onFulfilled,onRejected)

1.onFulfilled 和 onRejected 都是函数，如果不是函数，则忽略之。
2.onFulfilled ：resolve(成功)时调用 onFulfilled，必须在promise fulfilled后调用，且promise的value为其第一个参数，不能多次调用。
3.onRejected：reject(失败)时调用 onRejected，必须在promise rejected后调用， 且promise的reason为其第一个参数，不能被多次调用。

.catch只是 promise.then(undefined, onRejected) 的别名而已。

对于一个 promise，它的 then 方法可以调用多次。

Promise 是同步还是异步执行

再看个例子

Promise 是异步执行的。有个问题，为什么 setTimeout 模拟异步会比 promise 晚出现呢?

这就涉及到 microtask 和 macrotask 问题。简单说，promise 排在了异步下批执行的第一梯队，而setTimeout 则排在第二梯队。

Promise API

Promise.all 接收一个 promise对象的数组作为参数，当这个数组里的所有promise对象全部变为resolve或reject状态的时候，它才会去调用 .then 方法。

Promise.race 接收一个 promise对象的数组作为参数，只要有一个promise对象进入 FulFilled 或者 Rejected 状态的话，就会继续进行后面的处理。
